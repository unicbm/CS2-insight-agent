## 2. 下载与解压

### 第一步：下载压缩包

你会拿到一个名为 **`CS2-Insight-Agent-*版本号*.zip`** 的压缩文件。

### 第二步：解压文件

> [!CAUTION]
> **⚠️ 必须先解压！直接在压缩包里双击"启动.bat"是无法正常运行的！**

#### 如果你不知道怎么解压：

<details>
<summary>👉 点击展开：手把手教你解压</summary>

**方法 A：使用 Windows 自带功能（无需安装任何软件）**

1. 找到你下载的 `CS2-Insight-Agent-*版本号*.zip` 文件
2. **右键点击**这个文件
3. 选择 **「全部解压缩...」**（Windows 11 可能需要先点「显示更多选项」）
4. 在弹出的窗口中选择你想放的位置（比如桌面或 D 盘）
5. 点击 **「提取」** 按钮
6. 等待解压完成

**方法 B：使用 7-Zip / WinRAR（如果你电脑上有的话）**

1. **右键**点击 `CS2-Insight-Agent-*版本号*.zip`
2. 选择 **「解压到 CS2-Insight-Agent-*版本号*\」**
3. 等待完成即可

</details>

### 第三步：确认文件结构

解压完成后，你应该看到这样一个文件夹：

```
CS2-Insight-Agent-*版本号*/
├── 启动.bat           ← 双击这个启动程序！
├── README-使用说明.txt
├── backend/            ← 后端程序（不用管）
├── web/                ← 前端界面（不用管）
└── python/             ← Python 运行环境（不用管）
```

> [!IMPORTANT]
> 请确保你解压到的路径中**不包含中文字符**。例如：
> - ✅ `D:\CS2-Insight-Agent-*版本号*\`
> - ✅ `C:\Users\YourName\Desktop\CS2-Insight-Agent-*版本号*\`
> - ❌ `D:\游戏工具\CS2洞察\` ← 中文路径可能导致问题

## 3. 首次启动

1. 打开解压后的文件夹
2. **双击 `启动.bat`**
3. 会弹出一个黑色的命令行窗口，显示 `CS2 Insight 已启动`
4. 浏览器会自动打开页面 **`http://127.0.0.1:8000`**

> [!NOTE]
> - 黑色窗口**不要关闭**！关掉程序就停了
> - 如果浏览器没有自动打开，手动在浏览器地址栏输入 `http://127.0.0.1:8000` 即可
> - 如果提示端口被占用，打开 `启动.bat`（右键 → 编辑），把 `CS2_INSIGHT_PORT=8000` 改成 `8010` 或其他端口

打开后你将看到一个 **深色电竞风格** 的仪表盘界面。左侧是配置栏，中间是主工作区。

## 4. 配置 OBS（录制功能必需）

> 如果你只想 **解析 Demo 看高光片段列表**，可以跳过这一节。
>
> 如果你想让程序 **自动录制高光视频**，则必须配置 OBS。

### 第一步：安装 OBS Studio

如果你还没有安装 OBS：

1. 打开 [OBS 官方网站](https://obsproject.com/download) 或者到Steam中下载OBS最新版
2. 点击 **Windows** 下载按钮
3. 安装时一路默认「下一步」即可

### 第二步：开启 OBS WebSocket 服务

CS2 Insight 通过 OBS 的 WebSocket 接口来远程控制 OBS 的录制功能。你需要在 OBS 里开启它：

1. 打开 **OBS Studio**
2. 点击顶部菜单栏 **「工具」→「WebSocket 服务器设置」**

   > 如果你找不到这个选项，说明你的 OBS 版本太旧。请到官网下载最新版 OBS（28.0 以上自带 WebSocket）。

3. 在弹出的窗口中：
   - ✅ **勾选** 「启用 WebSocket 服务器」
   - **服务器端口**：默认 `4455`（不用改）
   - **密码**：
     - 你可以点「生成密码」自动生成一个，也可以自己设一个
     - **记住这个密码！** 后面要填到 CS2 Insight 里

4. 点 **「确定」** 保存

> [!TIP]
> 如果你嫌麻烦，也可以取消勾选「启用认证」来去掉密码，这样 CS2 Insight 里密码留空就行。但出于安全考虑不推荐这么做。

### 第三步：在 OBS 中添加「游戏捕获」来源 （可选，这一步为自动操作，现在程序会自动在OBS中创建名为CS2 Insight Recording的场景）

为了让 OBS 能录到 CS2 的画面：

1. 在 OBS 主界面底部的 **「来源」** 区域，点击 **「+」** 号
2. 选择 **「游戏捕获」** （**「显示器采集」**也可以，但是要注意可能录制过程中会被其他画面影响）
3. 取一个名字（比如"CS2"），点确定
4. 在弹出的属性窗口中：
   - **模式**：选择「捕获特定窗口」
   - **窗口**：选择 `[cs2.exe]: Counter-Strike 2`（如果 CS2 没在运行就先打开一次 CS2）
5. 点 **「确定」**

> [!IMPORTANT]
> **请确保 OBS 在整个录制过程中保持打开状态。** CS2 Insight 的自动录制功能会通过 WebSocket 远程控制 OBS 的「开始录制」和「停止录制」。

### 第四步：在 CS2 Insight 中填写 OBS 连接信息

回到 CS2 Insight 的浏览器页面：

1. 在左侧配置栏找到 **「OBS 连接」** 区域
2. 填写：
   - **地址 (Host)**：`localhost`（本机不用改）
   - **端口 (Port)**：`4455`（和 OBS 里一致）
   - **密码 (Password)**：填写你刚才在 OBS 里设的那个密码
3. 点击 **「测试连接」** 按钮

如果显示 ✅ 连接成功，恭喜你，OBS 配置完成！

如果连接失败，请检查：
- OBS 是否已打开且 WebSocket 已启用
- 端口号和密码是否填对了
- Windows 防火墙是否拦截了端口（一般本机通信不会）

## 5. 配置 AI 模式（可选）

AI 模式可以让大模型为你的每个高光/下饭片段生成 **评分 (0-100分)** 和 **锐评文案**。

不开启 AI 模式也能正常使用所有功能，只是看不到 AI 点评。

### 支持的 AI 服务商

| 服务商 | 推荐模型 | 价格 |
|--------|---------|------|
| **DeepSeek** (推荐) | `deepseek-chat` | 极便宜，几毛钱一场 |
| OpenAI | `gpt-4o-mini` | 较贵 |
| 其他 OpenAI 兼容 API | 各厂商自选 | 视厂商而定 |

### 获取 DeepSeek API Key（推荐）

1. 打开 [DeepSeek 开放平台](https://platform.deepseek.com/)
2. 注册/登录账号
3. 进入控制台 → **API Keys** → **创建新的 API Key**
4. 复制生成的 Key（以 `sk-` 开头的一串字符）
5. 充值几块钱就够用很久了

### 在 CS2 Insight 中配置 AI

1. 在左侧配置栏，将 **模式开关** 切换为 **「AI 洞察模式 (Pro)」**
2. 在展开的 LLM 配置面板中填写：
   - **服务商**：选择 `DeepSeek`
   - **模型**：`deepseek-chat`
   - **API Key**：粘贴你刚才复制的 Key
3. 配置会自动保存

## 6. 核心玩法：解析 Demo

### 什么是 Demo 文件？

Demo 就是 CS2 的比赛录像文件（`.dem` 格式）。你可以从以下途径获取：

- **Steam 个人比赛历史**：CS2 主菜单 → 你的比赛 → 点击某场比赛 → 最下方的「下载」按钮
- **5E / 完美平台**：对应平台的比赛详情页一般也有下载入口
- **HLTV / 比赛网站**：如果你要分析职业比赛

### 方式一：拖拽上传

1. 将 `.dem` 文件 **直接拖到** 页面中央的上传区域
2. 等待文件上传与预解析（会显示地图和玩家列表）
3. 在玩家列表中 **选择你要分析的玩家**（通常选你自己）
4. 点击 **「开始解析」**
5. 稍等片刻（通常 10-30 秒），结果会以 **卡片列表** 的形式展示

### 方式二：Demo 库（推荐长期使用）

如果你想持续管理大量 Demo，可以使用 **Demo 库** 功能：

1. 在左侧配置栏的 **「Demo 监听目录」** 中，添加你存放 Demo 文件的文件夹路径
2. 程序会自动扫描该目录下的所有 `.dem` 文件并加入库中
3. 在主界面的 Demo 库列表中选择想分析的 Demo
4. 选择目标玩家后点击解析

### 解析结果怎么看

每个被标记的片段会以卡片形式展示：

- **🔥 高光 (Highlight)**：该回合你杀了 3 个人或更多
- **💀 下饭 (Fail)**：被电击枪击杀 / 被沙鹰爆头 / 被队友击杀等名场面
- 如果开启了 AI 模式，还会显示：
  - **AI 评分**：0-100 分（分越高越精彩或越下饭）
  - **AI 锐评**：一段犀利又幽默的点评文案

## 7. 一键自动录制

这是 CS2 Insight 最硬核的功能——全自动控制 CS2 回放 + OBS 录制。

### 前提条件

- ✅ OBS 已配置并连接成功（参见 [第 4 节](#4-配置-obs录制功能必需)）
- ✅ CS2 已安装，且左侧配置栏中的 **CS2 路径** 正确指向 `cs2.exe`
  - 程序会自动探测 Steam 安装的 CS2 路径
  - 如果没有自动检测到，需要手动填写，例如：
    `D:\steam\steamapps\common\Counter-Strike Global Offensive\game\bin\win64\cs2.exe`
- ✅ OBS 保持打开状态

### 操作步骤

1. 先完成 Demo 解析，得到高光/下饭片段列表
2. **勾选** 你想录制的片段（或全选）
3. 点击底部的 **「🎬 一键录制」** 按钮
4. 程序会自动完成以下所有操作：
   - 🎮 自动启动 CS2，并加载 Demo 录像
   - ⏩ 自动跳转到每个片段对应的游戏时间点
   - 🎥 自动控制 OBS 开始录制
   - ⏱️ 等待片段播放完毕
   - ⏹️ 自动控制 OBS 停止录制
   - 🔁 自动跳到下一个片段，重复上述过程
   - 🚪 全部录完后自动关闭 CS2

> [!WARNING]
> **录制过程中请勿操作电脑！** 程序需要向 CS2 窗口发送键盘指令来控制 Demo 回放。录制期间切换窗口或打字可能导致命令丢失。
>
> 建议泡杯咖啡☕等几分钟，让程序自动完成一切。

### 录制文件在哪里

录制好的视频文件保存在 **OBS 的默认录制路径** 中。你可以在 OBS 中通过 **「文件 → 显示录像」** 快速打开。

也可以在 OBS 中通过 **「设置 → 输出 → 录像」** 查看和修改保存路径。

## 8. Demo 库功能

### 监听目录

在左侧配置栏中设置 **「Demo 监听目录」** 后，程序可以扫描该目录下的所有 Demo 文件。

> [!TIP]
> 可以设置多个监听目录。例如把 Steam 的 CS2 replay 目录和 5E 的下载目录都加进去。

### 关注名单

如果你在左侧配置了 **「关注玩家名单」**（填入你和你朋友的游戏昵称），那么当 Demo 库中有新文件进入时，程序会自动识别名单上的玩家，并在库列表中展示他们的 K/D/A 信息，方便你快速筛选想看的录像。

## 9. 常见问题 FAQ

<details>
<summary><b>Q：页面打不开 / 显示无法连接？</b></summary>

- 确保黑色命令行窗口没有关闭
- 尝试手动在浏览器输入 `http://127.0.0.1:8000`
- 检查是否有防火墙/杀毒软件拦截了端口 `8000`
- 尝试换一个端口：
**1.**编辑 `启动.bat`
把**第20行**的`http://127.0.0.1:8000`
和**第28行**的 `http://127.0.0.1:8000`两个`8000`改成 `8010`
**2.**CS2-insight-agent\backend\app目录下的`run_server.py`文件使用记事本打开，第64行的
`port = int(os.environ.get("CS2_INSIGHT_PORT", "8000"))`把`8000`改成`8010`
**这两处均需要修改。**

</details>

<details>
<summary><b>Q：OBS 测试连接失败？</b></summary>

- 确认 OBS 已经打开并且 WebSocket 服务器已启用（工具 → WebSocket 服务器设置）
- 检查端口号（默认 `4455`）与密码是否匹配
- 确认 OBS 版本 ≥ 28.0（旧版需要单独安装 obs-websocket 插件）
- 尝试在 OBS 中关闭认证（取消勾选），密码留空

</details>

<details>
<summary><b>Q：CS2 路径没有自动检测到？</b></summary>

手动找到 CS2 的安装路径。默认 Steam 安装一般在：
```
C:\Program Files (x86)\Steam\steamapps\common\Counter-Strike Global Offensive\game\bin\win64\cs2.exe
```
或者如果你把 Steam 游戏库装在了其他盘（比如 D 盘）：
```
D:\steam\steamapps\common\Counter-Strike Global Offensive\game\bin\win64\cs2.exe
```
在左侧配置栏的「CS2 路径」中填入完整路径。

> [!CAUTION]
> **⚠️⚠️⚠️ 需要特别注意⚠️⚠️⚠️**
> **为了保证录制以及还原玩家的原有设置，点击录制开始后请注意千万不要手动干预录制过程，也就是说最好鼠标键盘都不要再动了！**
> **若中途有干预录制或者控制台输入，可能会发生把玩家本机的游戏设置覆盖的问题！**

</details>

<details>
<summary><b>Q：实验性功能 POV HUD 是什么？</b></summary>

- 在侧边栏打开「常用参数管理」，底部「实验性功能」中可勾选 **POV**。开启后，本地 Demo 回放录制时会临时安装项目自带的 `pov.vpk`，并增量修改 CS2 `game/csgo/gameinfo.gi` 的搜索路径；**录制结束后程序会自动恢复**。
- 开启 POV 后，部分与 HUD / 雷达相关的预热选项会被隐藏并由程序强制设定；不要用于连接官方匹配服务器。
- 若异常退出导致修改未恢复，可在同一页面查看提示并点击「恢复 POV 修改」（需先关闭 CS2）。

</details>

<details>
<summary><b>Q：录制时 CS2 没有画面 / OBS 录到黑屏？</b></summary>

- 确保 OBS 的「游戏捕获」来源设置正确（捕获 `cs2.exe` 的窗口）
- 尝试在 OBS 的「游戏捕获」属性中把模式改为「捕获任何全屏应用程序」
- 确认 CS2 没有被最小化到后台
- 如果使用了多显示器，确保 CS2 和 OBS 捕获的是同一个屏幕

</details>

<details>
<summary><b>Q：成片有黑边？</b></summary>

- 你的CS2是4:3显示时，有可能发生该情况
- 确保在OBS 的「游戏捕获」中右键->变换->拉伸到全屏

</details>

<details>
<summary><b>Q：发布了新版本如何更新？</b></summary>

- 先将老版本的程序关闭，再将新版本的压缩包全部解压到覆盖到老版本中，.json以及.db文件不要动，那是你的配置及demo数据

</details>

<details>
<summary><b>Q：最终效果把控制台录进去了？</b></summary>

- OBS 设置中的输出-录像质量，不能选与串流画质相同

</details>

<details>
<summary><b>Q：录制时 CS2 停留在游戏主菜单？</b></summary>

- 目前最新的CS2版本仅能播放4月21日以后的demo 也就是ag2动作系统更新以后的demo 如果想录制iem里约这种更新前比赛的镜头需要把cs2版本回退到1.41.4.1

</details>

<details>
<summary><b>Q：录制出来的视频是第三人称视角？</b></summary>

程序会自动切换成第一人称视角来观看目标玩家。如果出现问题，可以尝试在解析结果中确认选择的目标玩家名称与 Demo 中的名称完全一致。

</details>

<details>
<summary><b>Q：解析很慢怎么办？</b></summary>

- 第一次解析会稍慢，后续会快很多
- 建议不要同时解析多个超大 Demo
- 如果你开启了 AI 模式，AI 点评需要网络请求，视 API 响应速度可能需要额外几秒

</details>

<details>
<summary><b>Q：AI 模式报错？</b></summary>

- 检查 API Key 是否正确（以 `sk-` 开头）
- 检查你的 API 账户是否还有余额
- 如果使用 DeepSeek，确认 Base URL 为 `https://api.deepseek.com`
- 不开启 AI 模式不影响其他所有功能

</details>

<details>
<summary><b>Q：我是用 5E / 完美平台打的比赛，Demo 文件在哪？</b></summary>

- **5E**：5E 客户端 → 比赛记录 → 选择某场比赛 → 下载 Demo
- **完美世界**：完美世界竞技平台 → 比赛记录 → 下载录像
- 下载的 `.dem` 文件可以直接拖入 CS2 Insight 使用

</details>

<details>
<summary><b>Q：可以同时录多场 Demo 的片段吗？</b></summary>

可以。程序支持批量录制——当你的待录制队列中包含多个 Demo 的片段时，程序会自动依次加载每个 Demo 并录制其中的选中片段。

</details>
